# OFGW PR Writing Guide

## Scope

This guide supports PR-facing writing for OFGW.

## Core Sections

- Purpose
- Change Summary
- Verification
- Follow-up Documents

## OFGW Rules

- Use PR terminology consistently in user-facing text
- State the touched modules explicitly
- Separate backend verification from frontend/admin verification
- If only `ofgwSrc` changed, do not imply `webterminal` or `ofgwAdmin` were also validated

## Useful References

- `agent/<topic>/analysis_report.md`
- `agent/<topic>/design_spec.md`
- `agent/<topic>/plan_tasks.md`
- `generated/manual/` when manual follow-up is required
