# 메일 작성 가이드

## 구조

```
제목: [<제품>/<IMS#XXXXXX or OFV7-XXXX>/<topic>] 핵심 내용 한 줄

본문:
안녕하세요.
{소속} {이름}입니다.

{목적 한 문장}

1. {섹션1 제목}
   - 내용

2. {섹션2 제목}
   - 내용

3. {섹션3 제목}
   - 내용

{마무리 한 문장}

감사합니다.

{이름} 드림
```

## 규칙

### 제목

- `[AIM/IMS#352569/ACS DCPRES] 분석 결과 공유` — 제품 태그 + 이슈 번호 + topic + 핵심
- 제목만 보고 메일 내용을 예측할 수 있어야 함

### 톤

- **격식체 필수**: "~합니다", "~드립니다", "~부탁드립니다"
- 인사: **"안녕하십니까"** (통일)
- 마무리: "감사합니다" + 이름 + "드림"

### 본문

- **번호 매기기** (1. 2. 3.)로 섹션 구분
- 구분선 (`━━━`, `---`, `===`) 사용 금지 — 비즈니스 메일에 부적절
- 각 섹션은 3~5줄 이내
- 전체 본문: 20줄 이내 권장

### 독자별 추상화

- **팀 내**: 모듈/함수 수준 OK. 단, 파일:라인 수준은 불필요
- **팀 외/고객**: 기능/동작 수준만. 코드 용어 금지
- **CC 포함 시**: CC 수신자 중 가장 넓은 독자에 맞춤

### 발신자

- 발신자 이름/소속은 사용자 정보를 사용
- 임의로 이메일 주소나 소속을 만들지 않음
- 서명 형식: `{이름} 드림\n{팀명} | {회사명}`

### 첨부

- 첨부가 필요하면 본문에 첨부 파일 목록 기술
- 첨부 없이 보낼 수 있으면 본문에 핵심만 포함

## 발송 전 체크

- [ ] 수신자(To/CC) 확인
- [ ] 제목이 내용을 반영하는가
- [ ] 격식체인가
- [ ] 구분선/장식 없는가
- [ ] **사용자 승인을 받았는가**
